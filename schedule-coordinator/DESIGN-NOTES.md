# 6명 회의 일정 잡기 — 설계 노트 (제출용)

> 방향: **주최자 경험에 집중**한 새 산출물. 네이버웍스 클론·플로팅 챗봇은 걷어내고,
> "6명 시간 찾기" 핵심 흐름만 3~4화면 고완성도로. 시각 완성도는 impeccable(재시작 후)로.

## 채점 기준 → 화면 매핑

| 기준 | 이 산출물에서 증명하는 지점 |
|---|---|
| 문제 정의 | 3문항 답변 Q1 + 히어로 화면이 "비었음≠가능함"을 시각적으로 드러냄 |
| 솔루션 설계 | 3단계 표현력·필참/선택·재실불확실·사람 최종결정이 흐름에 자연히 반영 |
| 시각적 완성도 | 익숙한 캘린더/가용그리드 패턴을 맥락에 맞게 다듬음 (impeccable 패스) |

## 화면 구성 (주최자 중심 3 + 참석자 입력 1)

0. **(참석자·보조)** 안 되는 시간만 빼기 — 점심 직후·외근 요일 제약이 시스템에 들어오는 출처.
   3단계(괜찮음/피하고싶음+이유/안됨) + 요일별 기본 근무(외근·재택) 1회 설정.
1. **(주최자)** 회의 만들기 — 제목·1시간·다음 주까지·6명 추가·필참/선택 지정.
2. **(주최자) ★히어로** — 6명의 4단계 상태가 한 타임라인에 겹쳐 보임 +
   필참 전원 가능·선호 높은 **추천 시간 + 이유** + 완벽X면 **선택지(범위 확장 / 양해 요청)**.
   코어타임 밖은 '재실 불확실'로 단정하지 않음.
3. **(주최자)** 확정 → 초대·알림 발송 (양해 필요 시 그 대상에게만 확인).

## 3문항 답변 초안

### Q1. 어떤 문제를 발견했고, 왜 문제인가
표면적 어려움은 "6명 빈 시간을 못 찾는 것"처럼 보이지만, 실제 비용은 **잡고 나서 다시 잡는 반복**에 있습니다.
주최자는 겹치는 빈칸에 일단 회의를 잡고, 초대를 보낸 뒤에야 "그 시간은 좀…"이라는 반응을 받습니다.
그때부터 재조율이 시작되고, 인원수가 늘수록 한 사람이 "별로예요" 할 때마다 처음부터 다시 시작됩니다.
근본 원인은 **표현력의 부재**입니다. 첫째, '비어있음'은 '가능함'이 아닙니다 — 같은 빈칸에도
'괜찮은 시간', '되지만 점심 직후라 피하고 싶은 시간', 유연근무라 '아직 출근 전이라 재실이 불확실한 시간'이
뒤섞여 있는데 기존 툴은 하나로 뭉갭니다. 둘째, 제약의 무게가 다릅니다 — '외근이라 안 되는 날'과
'피하고 싶은 시간', '꼭 와야 하는 사람'과 '빠져도 되는 사람'은 성격이 다른데 동일하게 취급됩니다.
결국 문제는 빈 시간을 못 찾는 게 아니라, **제약·선호가 드러나기 전에 성급히 정하고 어긋나면 다시
조율하는 반복 비용**이며, 그 뿌리는 이 맥락을 표현할 방법이 없다는 것입니다.

### Q2. 어떻게 해결했나
핵심은 시간을 "비었다/찼다" 이분법이 아니라 **얼마나 괜찮은지** 표현할 수 있게 만든 것입니다.
참석자는 최초 1회 요일별 기본 근무 가능 시간을 설정합니다(유연근무·외근·재택 반영: 월 10~19시,
수 재택 오후만 등). 그 위에 회의별로 시간대 선호를 **괜찮아요 / 되지만 피하고 싶어요(이유) / 안 돼요**
3단계로 표시합니다. 기본 설정은 한 번이면 이후 모든 조율에 자동 반영돼 반복 입력을 없앱니다.
주최자는 참석자를 **필참/선택**으로 구분해 등록합니다. 시스템은 단순히 전원 빈 시간이 아니라
각자의 근무 패턴·선호를 함께 계산해 **필참 전원 가능하면서 선호가 가장 높은 시간을 이유와 함께**
추천합니다. 코어타임 밖 빈칸은 '재실 불확실'로 두고 함부로 가능 처리하지 않습니다. 완벽히 맞는
시간이 없으면 시스템이 임의로 정하지 않고 **"양해 구하고 진행" / "범위 넓혀 다시 찾기"** 선택지를
주최자에게 제시해, 최종 판단은 사람이 내립니다.

### Q3. 왜 이 방식인가
이분법으로는 "비었지만 불편한 시간"과 "정말 편한 시간"을 구분할 수 없고, 그러면 일단 잡고 나중에
문제 터지는 사후 대응이 반복됩니다. 3단계로 나누면 처음부터 더 나은 선택지를 가려내 재조율 자체를
줄일 수 있습니다. 근무 패턴을 매번 입력하게 하면 번거로워 안 쓰게 되므로, 1회 설정 자동 반영으로
참여 장벽을 낮췄습니다 — 좋은 기능도 입력 비용이 크면 쓰이지 않습니다. 필참/선택 구분은 주최자가
이미 머릿속으로 하던 판단을 시스템에 옮긴 것으로, 비핵심 인원 한 명 때문에 전체가 막히는 비효율을
막습니다. 마지막으로 6명을 100% 만족시키는 시간은 현실에 드뭅니다. 시스템이 임의로 정하면 결정의
책임과 맥락을 주최자가 잃으므로, 근거 있는 선택지를 제시하고 최종 결정은 사람이 하도록 설계해
신뢰를 유지했습니다.
