# 일정 조율 도우미 (Schedule Coordinator)

자율출근 팀을 위한 **회의 시간 조율 프로토타입**. 참석자·소요 시간·기간만 주면, 각자의 근무 패턴·캘린더·선호도를 반영해 이유와 함께 회의 시간을 추천합니다.

## 화면 (프로토타입)

`calendar-prototype.html` (= `index.html`) 한 파일에 3개 화면이 들어 있고, 좌하단 **데모 네비게이션**으로 전환합니다.

- **주최자·캘린더** — 내 캘린더 뷰
- **일정 작성** — 회의 생성 폼 + 우측 `일정 조율 도우미` 챗
  - 챗에서 `@` 입력 → 참석자 자동완성(멘션), 멘션한 사람이 그대로 필참으로 반영
  - 참석자 근무 패턴·캘린더·선호도를 반영해 후보 시간 랭킹 + **이유와 함께 추천**
  - 피하고 싶다고 표시한 사람에겐 **정중한 슬랙 양해 요청** 메시지(양해 톤·거절 쉬움) 시뮬레이션
- **참석자·초대** — 초대받은 사람 화면 (`안 되는 시간만 빼주세요`)
  - **확인 중심 UX**: 입력이 기본이 아니라 확인이 기본
    - 캘린더의 일정·휴가·외근은 **자동 제외**
    - `내 근무 시간` **1회 설정**(자율출근이면 본인 시간대 · 고정 아님) → 근무 외 시간은 그리드에서 자동 회색 처리
    - **말로 빼기** 한 줄 입력 (예: "이번 주 오전만 빼줘") → 그리드 자동 반영
    - 세밀 조정용 그리드는 **접힘**(`직접 시간 조정하기`), 예외 있을 때만 펼침
    - 대부분 **0~1클릭**으로 `✅ 다 괜찮아요 · 이대로 보내기`

- **six-way-scheduler.html** — 자율출근 6명의 재실 시간대 겹침 분석 시각화(확실/불확실/점심 드리프트)

## 설계 원칙

- **노력은 최초 1회, 매번은 무입력이 기본** — 근무 패턴을 한 번 등록해두면 회의마다 다시 찍지 않아도 됨
- **자동값 + 캘린더가 진실의 소스**, 수동 입력은 예외 미세조정
- **자율출근 반영** — 근무시간은 "대체로 그렇다(확률)"로 다루고, 실제 캘린더 일정이 항상 우선

## 실행

빌드 없음. 브라우저로 `index.html` 열기.

```bash
open index.html
```

## 정보 보호

참석자가 표시한 시간·메모는 "모두에게 맞는 시간을 찾는 데만" 사용하며, 누가 무엇을 표시했는지는 개별로 드러나지 않는 것을 전제로 설계.
